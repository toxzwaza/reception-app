# Akioka-Local-CA 証明書 インストール手順

受付システム（reception-app）でスキャナー／プリンター（192.168.210.x）に接続する際、証明書警告が出ないようにするための社内ルート認証局（CA）証明書。

## このフォルダの内容

| ファイル | 形式 | 用途 |
|---------|-----|------|
| `Akioka-Local-CA.crt` | DER/PEM | Windows・Android で使用 |
| `Akioka-Local-CA.pem` | PEM | Linux・Mac・iOS 等で使用（中身は同じ） |

## 証明書情報

| 項目 | 値 |
|------|-----|
| Subject / Issuer | `CN=Akioka-Local-CA, O=Akioka, OU=DX, L=Kurashiki, ST=Okayama, C=JP` |
| 有効期間 | 2025-11-19 〜 **2035-11-17（10年）** |
| 用途 | 192.168.210.90（スキャナー）・192.168.210.91（プリンター）のSSL |

## インストール方法

### Windows（Chrome / Edge / IE 共通）

1. `Akioka-Local-CA.crt` をダブルクリック
2. 「証明書のインストール」をクリック
3. 保存場所: **「ローカル コンピューター」** を選択（管理者権限必要）※個人のみでOKなら「現在のユーザー」
4. 「証明書をすべて次のストアに配置する」を選択
5. 参照 → **「信頼されたルート証明機関」** を選択
6. 「次へ」→「完了」
7. セキュリティ警告 → 「はい」
8. **ブラウザを再起動**

#### 確認方法（Windows）
```
certmgr.msc を開く
→ 信頼されたルート証明機関 → 証明書
→ 「Akioka-Local-CA」があれば成功
```

### Mac（Safari / Chrome）

1. `Akioka-Local-CA.pem` をダブルクリック → キーチェーンアクセスが開く
2. **「システム」キーチェーン** を選択して追加
3. キーチェーンアクセス → 「Akioka-Local-CA」をダブルクリック
4. 「信頼」セクションを開く
5. 「この証明書を使用するとき」→ **「常に信頼」** に変更
6. パスワード入力で確定
7. **ブラウザを再起動**

### Firefox（Windows/Mac共通・独自ストア）

Firefox は OS の証明書ストアを使わないため別途設定：

1. Firefox → 設定（about:preferences）
2. 「プライバシーとセキュリティ」→「証明書」→「証明書を表示」
3. 「認証局証明書」タブ → 「インポート」
4. `Akioka-Local-CA.crt` を選択
5. **「この認証局によるウェブサイトの識別を信頼する」** にチェック
6. OK → Firefox再起動

### iPhone / iPad

1. `Akioka-Local-CA.crt` をメール等で iPhone に送る or AirDrop
2. 添付をタップ → 「プロファイルがダウンロードされました」
3. 設定 → 一般 → **VPNとデバイス管理** → ダウンロード済みプロファイル
4. 「Akioka-Local-CA」をタップ → インストール → パスコード入力
5. **さらに**: 設定 → 一般 → 情報 → **証明書信頼設定**
6. 「Akioka-Local-CA」を **オンにする**（重要！）
7. Safari/Chrome を再起動

### Android（Chrome）

1. `Akioka-Local-CA.crt` を Android にダウンロード
2. 設定 → セキュリティ → 暗号化と認証情報 → 「CA証明書をインストール」
   （機種により: 設定 → セキュリティ → その他のセキュリティ設定 → 証明書のインストール）
3. ダウンロード済みの `Akioka-Local-CA.crt` を選択
4. 証明書名を「Akioka-Local-CA」のまま確定
5. 用途: **VPN とアプリ** を選択
6. Chrome を再起動

## インストール確認方法

CA設定後、以下のURLにアクセスして **鍵マークが表示されれば成功**：

- スキャナー管理画面: https://192.168.210.90:5001/video_feed
- プリンター管理画面: https://192.168.210.91:5000/

証明書警告が出ずに画面が表示されたらOK。

## トラブルシューティング

### まだ警告が出る
- **ブラウザ完全再起動**（全ウィンドウ閉じる）
- プライベートウィンドウで試す（キャッシュ回避）
- 証明書ストアを開いて Akioka-Local-CA が「信頼されたルート」にあるか確認

### iPhone で承認後もエラー
- 「証明書信頼設定」でオンにしていない可能性
- 設定 → 一般 → 情報 → 証明書信頼設定 を再確認

### 新しいPCでの初期セットアップ
新入社員・新端末のセットアップ時、受付システムを使う場合は必ずこの CA をインストールする運用にする。

## 技術的背景

- RPi のサーバー証明書は Akioka-Local-CA（社内私設CA）で署名されている
- ブラウザは標準では社内CAを知らないため信頼されない → エラー
- Akioka-Local-CA をルートCAとして登録すると、このCAが発行する全ての証明書（192.168.210.90, 192.168.210.91 等）が自動的に信頼される
- 将来新しい機器を追加する際も、同じCAで証明書を発行すれば再配布不要
