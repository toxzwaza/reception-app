# 新端末セットアップ手順 — アキオカ受付システム

> 最終更新: 2026-04-16
> 対象: 受付システム（reception-app）を新しく利用する端末のセットアップ

---

## 概要

新しい端末（受付タブレット・事務所PC・スマホ等）でアキオカ受付システム（https://akioka-reception.cloud/）を利用する際の初期セットアップ手順。

### セットアップ全体の流れ

1. **ネットワーク接続** — 社内Wi-Fiに接続
2. **Akioka-Local-CA 証明書インストール** — スキャナー/プリンター連携のため必須
3. **ブラウザで受付システムにアクセス** — 動作確認
4. **ブックマーク / ホーム画面追加**（任意）

---

## 事前準備（システム管理者側）

以下のファイルを端末に転送できるように準備：

| ファイル | 保管場所 |
|---------|---------|
| Akioka-Local-CA.crt | `reception-app/RASPBERRY_PI/certs/Akioka-Local-CA.crt` |
| Akioka-Local-CA.pem | `reception-app/RASPBERRY_PI/certs/Akioka-Local-CA.pem` |

転送方法の例：
- メール添付
- USBメモリ
- Teams / OneDrive でファイル共有
- QRコード経由のクラウドリンク

# 【Windows】セットアップ手順

対象: Windows 10 / 11（Chrome / Edge 利用を想定）

## 1. Wi-Fi接続

社内ネットワーク（192.168.210.x）に接続する。

## 2. Akioka-Local-CA 証明書のインストール

### 手順

1. 管理者から受け取った **`Akioka-Local-CA.crt`** をデスクトップに保存
2. ファイルを **右クリック → 「証明書のインストール」**
3. **保存場所**: 「ローカル コンピューター」を選択（※管理者権限が必要）
   - 個人利用のみでOKなら「現在のユーザー」でも可
4. 「次へ」
5. 「**証明書をすべて次のストアに配置する**」を選択
6. 「参照」をクリック → **「信頼されたルート証明機関」** を選択
7. 「次へ」→「完了」
8. セキュリティ警告 → 「はい」
9. 「正しくインポートされました」→ 「OK」

### 確認

`Win + R` → `certmgr.msc` と入力して実行。

「信頼されたルート証明機関」→「証明書」 の中に **「Akioka-Local-CA」** があれば成功。

## 3. ブラウザで受付システムにアクセス

**⚠ ブラウザを完全再起動**（全ウィンドウ閉じてから開く）してから：

1. Chrome または Edge を起動
2. アドレスバーに `https://akioka-reception.cloud/` を入力
3. トップ画面が表示される
4. URL バー左に **🔒 鍵マーク** が表示されていればOK

### 動作確認（スキャナー/プリンター利用の場合）

1. **納品・集荷** → **納品** → **納品書** を選択
2. スキャン画面でカメラ映像が表示されればOK
3. 警告が出る場合は CA インストール漏れ → 2. の手順を再確認

## 4. ブックマーク / PWA インストール

### ブックマーク追加

`Ctrl + D` → ブックマークバーに追加

### PWA としてインストール（Chrome推奨）

1. アドレスバー右端の **「インストール」アイコン（⊕ マーク）** をクリック
2. 「インストール」を選択
3. デスクトップ / スタートメニューに受付システムのアイコンが追加される
4. 独立したアプリケーションウィンドウで起動可能

# 【Mac】セットアップ手順

対象: macOS 12 以降（Safari / Chrome 利用を想定）

## 1. Wi-Fi接続

社内ネットワーク（192.168.210.x）に接続する。

## 2. Akioka-Local-CA 証明書のインストール

### 手順

1. 管理者から受け取った **`Akioka-Local-CA.pem`** をダウンロード
2. **`Akioka-Local-CA.pem` をダブルクリック**
3. 「キーチェーンアクセス」が起動
4. 追加先: **「システム」キーチェーン** を選択 → 追加
5. Macのパスワード入力

### 信頼設定（重要）

1. キーチェーンアクセスで **「Akioka-Local-CA」** を検索
2. **ダブルクリック** で詳細を開く
3. **「▼ 信頼」** セクションを展開
4. 「この証明書を使用するとき」→ **「常に信頼」** に変更
5. ウィンドウを閉じる → パスワードで確定

### 確認

キーチェーンアクセスで「Akioka-Local-CA」に **青い十字マーク** が付いていればOK。

## 3. ブラウザで受付システムにアクセス

**⚠ Safari / Chrome を完全終了**（Cmd + Q）してから再起動：

1. Safari または Chrome を起動
2. `https://akioka-reception.cloud/` にアクセス
3. URL バー左に **🔒 鍵マーク** が表示されていればOK

### Firefox を使う場合

Firefox は OS のキーチェーンを使わない独自ストアのため、別途設定必要：

1. Firefox → 環境設定 → プライバシーとセキュリティ
2. 下部の「証明書を表示」
3. 「認証局証明書」タブ → 「インポート」
4. `Akioka-Local-CA.crt` を選択
5. **「この認証局によるウェブサイトの識別を信頼する」** にチェック
6. OK → Firefox 再起動

## 4. ブックマーク / Dock 追加

### ブックマーク

`Cmd + D` でブックマーク追加

### Dock に追加（PWA）

Chrome の場合:
1. アドレスバー右端の **「インストール」アイコン（⊕）** をクリック
2. Dock にアプリとして表示される

Safari の場合:
1. ページを表示
2. 「共有」→ 「Dock に追加」

# 【Android】セットアップ手順

対象: Android 10 以降（Chrome 利用を想定）

## 1. Wi-Fi接続

社内Wi-Fi に接続する。

## 2. Akioka-Local-CA 証明書のインストール

### 証明書ファイルの受け取り方

- メール添付で受信 → 添付ファイルをダウンロード
- AirDrop 類似機能で受信
- Google Drive / Teams などから端末にダウンロード

### インストール手順（機種により多少異なる）

#### Android 12 以降の標準手順

1. **設定** を開く
2. **「セキュリティとプライバシー」** → **「その他のセキュリティ設定」**
   - 機種により: 「セキュリティ」→「暗号化と認証情報」
3. **「CA証明書をインストール」** をタップ
4. セキュリティ警告が表示される → 「インストール」をタップ（警告は正常）
5. ファイル選択画面で **`Akioka-Local-CA.crt`** を選択
6. インストール完了

#### Android 11 以前の手順

1. 設定 → **「生体認証とセキュリティ」** →「その他のセキュリティ設定」
2. 「デバイス管理アプリ」→「証明書の管理」→「ストレージからインストール」
3. `Akioka-Local-CA.crt` を選択
4. 証明書名: **「Akioka-Local-CA」** のまま確定
5. 用途: **「VPNとアプリ」** を選択

### 確認

設定 → セキュリティ → 信頼済みの認証情報 → **「ユーザー」タブ** に「Akioka-Local-CA」があれば成功。

## 3. Chrome で受付システムにアクセス

**⚠ Chrome を完全終了**（タスクから削除）してから：

1. Chrome を起動
2. `https://akioka-reception.cloud/` にアクセス
3. URL バー左に **🔒 鍵マーク** が表示されていればOK

**注意**: Android では「この接続は安全ではありません」という簡易警告が表示される場合がありますが、**ユーザーインストールの証明書使用時の通常表示**です。鍵マークが出ていれば通信は暗号化されています。

## 4. ホーム画面追加 / PWA インストール

1. ページを開いた状態で **右上の ⋮ メニュー** をタップ
2. 「**ホーム画面に追加**」または「**アプリをインストール**」をタップ
3. 名前を確認して「追加」
4. ホーム画面にアイコンが追加される

タブレット受付端末として使う場合、このアイコンからフルスクリーン起動可能。

## セットアップ後の動作確認

すべてのOS共通で、以下を確認して問題なければ完了：

| 確認項目 | 期待結果 |
|---------|---------|
| ① https://akioka-reception.cloud/ にアクセス | トップ画面表示 + 鍵マーク |
| ② 納品 → 納品書 へ進む | スキャナー映像が表示される |
| ③ QRコード印刷（管理者操作時） | ラベルプリンターから印刷される |

## トラブルシューティング

### 「保護されていない通信」と表示される

- ブラウザを完全再起動していない → 全ウィンドウ閉じて再起動
- CA インストールでストア指定を間違えた → やり直し
- iOS の場合「証明書信頼設定」で有効化忘れ

### スキャナーのカメラ映像が映らない

- 証明書が未信頼 → 別タブで `https://192.168.210.90:5001/video_feed` を開いて確認
- スキャナーRPi側が停止 → システム管理者に連絡

### PWA インストールできない

- HTTPS 通信で鍵マークが出ていないとインストール不可 → CA設定を見直す

問題が解決しない場合は **システム開発部門** までお問い合わせください。
