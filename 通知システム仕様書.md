# アキオカ受付システム 通知システム仕様書

> 最終更新: 2026-04-16
> 対象: reception-app 全通知機能の現状整理

---

## 1. 全体像

本システムは **5種類のトリガーイベント** に対して、**3種類の通知手段**（電話 / メール / Teams）を組み合わせて担当者に通知する仕組み。

### トリガーイベント一覧

| トリガーID | 発火タイミング | 実装状態 |
|----------|--------------|---------|
| `visitor_checkin` | 来訪者チェックイン時（アポイント・面接） | ✅ |
| `delivery_received` | 納品書受信時 | ✅ |
| `pickup_received` | 集荷伝票受信時 | ✅ |
| `interview_call` | 面接時通話（現状 visitor_checkin で兼用） | ⚠️ 未使用 |
| `appointment_reminder` | アポイント前リマインダー | 🚧 未実装 |

### 通知手段一覧

| 種別 | 実装 | 備考 |
|------|-----|------|
| **Teams** | ✅ 完全動作 | Adaptive Card + メンション、Webhook経由 |
| **電話 (Twilio)** | ⚠️ 一部動作 | 面接画面の自動発信のみ。他のトリガーはログ出力のみ |
| **メール** | 🚧 ログのみ | `AppointmentConfirmation` クラスのみ実装、呼び出し未結線 |

## 2. データベース構造

### 2.1 notification_settings（通知設定）

1トリガーにつき1以上の設定レコードを持つ。管理画面から作成・編集可能。

| カラム | 型 | 用途 |
|-------|-----|------|
| id | bigint PK | |
| name | varchar(255) | 設定名（例: "面接時通話"） |
| description | varchar(255) | 説明（任意） |
| **trigger_event** | varchar(255) | トリガー ID（上記一覧） |
| is_active | boolean | 有効/無効 |
| settings | json | 追加設定（auto_call/call_delay 等） |

### 2.2 notification_recipients（通知受信者）

1つの通知設定に対して複数の受信者を登録できる。各受信者は1つの通知手段を持つ。

| カラム | 型 | 用途 |
|-------|-----|------|
| id | bigint PK | |
| notification_setting_id | bigint FK | 親設定 (cascade delete) |
| user_id | bigint | users.id |
| **notification_type** | enum | `phone` / `email` / `teams` |
| **notification_data** | varchar(255) | 電話番号 / メールアドレス / Teams メンションID |
| is_active | boolean | 有効/無効 |

### 2.3 interview_phones（レガシー、現状未使用方向）

面接担当者の電話番号を登録していた独立テーブル。現在は `notification_recipients` で一元管理する方針。互換性のためマイグレーションは残存。

## 3. 通知サービスクラス

### 3.1 NotificationService

`app/Services/NotificationService.php` (約600行)

**汎用の通知ハブ**。`trigger_event` を受けて、notification_settings から関連する受信者を全て取得し、各受信者の `notification_type` に応じて振り分ける。

#### 主要メソッド

| メソッド | 用途 |
|---------|------|
| `sendNotification($trigger, $data)` | 汎用送信（delivery/pickup/アポイント用） |
| `sendAppointmentNotification($trigger, $data, $mentionData)` | 来訪者チェックイン専用（メンション情報付き） |
| `sendToRecipient($recipient, $data)` | 1受信者への送信ディスパッチ |
| `sendPhoneNotification()` | **ログ出力のみ**（Twilio REST未結線）|
| `sendEmailNotification()` | **ログ出力のみ**（未実装）|
| `sendTeamsNotification()` | → `sendTeamsMention()` で Webhook POST |
| `sendTeamsMention($webhookUrl, $mentionName, $mentionId, $msg)` | Adaptive Card で送信 |

### 3.2 TeamsNotificationService

`app/Services/TeamsNotificationService.php` (約520行)

**Teams 専用の特化サービス**。上記 NotificationService の Teams 処理も内部で利用。

#### 主要メソッド

| メソッド | 用途 | 呼び出し元 |
|---------|------|----------|
| `sendCheckinNotification($visitor)` | 来訪者チェックイン通知 | VisitorController |
| `notifyInterviewArrival()` | 面接到着通知（担当者にメンション） | InterviewController |
| `notifyDepartmentVisitor($visitor, $groupId)` | 部署の全員にメンション | OtherVisitorController |
| `getInterviewMentionIds()` | visitor_checkin の email type recipients 取得 | 内部用 |

メンション用の ID はnotification_recipientsの `email` type に格納されている emailアドレスを流用。

## 4. トリガー別フロー

### 4.1 来訪者チェックイン（アポイントあり）

```
受付タブレット
  ↓ QR読み取り / 受付番号入力
POST /appointment/check-in-qr
POST /appointment/check-in-number
  ↓
AppointmentController::checkInQr() / checkInNumber()
  ↓
Visitor::create()
  ↓
sendAppointmentNotification($visitor)
  ↓
NotificationService::sendAppointmentNotification('visitor_checkin', ...)
  ↓
notification_settings WHERE trigger_event='visitor_checkin' AND is_active=1
  ↓
各設定の activeRecipients:
  ├─ phone  → Log のみ（Twilio REST未結線）
  ├─ email  → Log のみ
  └─ teams  → Webhook POST（Adaptive Card メンション付）→ Teams 通知 ✅
```

### 4.2 来訪者チェックイン（アポイントなし）

```
受付タブレット
  ↓ 部署選択 → 訪問者情報入力
POST /other-visitor/store
  ↓
OtherVisitorController::store()
  ↓
Visitor::create(visitor_type='other', group_id=<部署>)
  ↓
TeamsNotificationService::notifyDepartmentVisitor($visitor, $groupId)
  ↓
部署内の User を全員取得 → 全員メンション
  ↓
Adaptive Card POST → Teams 通知 ✅
```

### 4.3 面接受付

```
受付タブレット
  ↓
GET /interview
  ↓
InterviewController::index()
  ├─ TeamsNotificationService::notifyInterviewArrival()
  │    ↓
  │    visitor_checkin の email type recipients を取得
  │    ↓
  │    担当者メンション付き Adaptive Card POST → Teams 通知 ✅
  │
  └─ getInterviewPhones()
       ↓
       visitor_checkin の phone type recipients を取得
       ↓
       Inertia で Interview/Index.vue に phoneNumbers を渡す
       ↓
       TwilioAutoCall.vue（ブラウザ内 Twilio Device SDK）
       ↓
       POST /twilio-voice/token → JWT 取得
       ↓
       Twilio → 本番サーバ /twilio-voice/outgoing → <Dial>
       ↓
       担当者スマホが鳴る ✅
```

### 4.4 納品・集荷受信

```
受付タブレット
  ↓ 書類スキャン
POST /delivery/store（または /pickup/store）
  ↓
DeliveryController::store() / PickupController::store()
  ↓
Delivery::create() / Pickup::create()
  ↓
NotificationService::sendNotification('delivery_received' | 'pickup_received', ...)
  ↓
notification_settings WHERE trigger_event=<該当> AND is_active=1
  ↓
各 recipient:
  ├─ phone → Log のみ
  ├─ email → Log のみ
  └─ teams → Webhook POST → Teams 通知 ✅
```

### 4.5 アポイント登録時のメール（別系統）

```
管理画面で新規アポイント登録
  ↓
AdminAppointmentController::store()
  ↓
Appointment::create() + QRコード生成
  ↓
AppointmentConfirmation メール送信（SMTP）
  ↓
visitor_email にメール到着 ✅（SMTP設定済みなら動作）
```

これは通知サービスを経由せず、Laravel 標準の Mailable を直接使用。

### 🔴 優先度高
1. **電話通知（Twilio REST API 発信）が未結線** — 面接以外のトリガーでも自動発信できるようにする

## 14. 運用・開発指針

### 追加開発の流れ（例: 新しいトリガーを追加する場合）

1. `NotificationSetting::TRIGGER_EVENTS` に定数追加
2. コントローラの該当メソッドから `NotificationService::sendNotification($trigger, $data)` を呼ぶ
3. `/admin/notification-settings` で新トリガー用の通知設定を作成、受信者を登録
4. テスト: `check_notifications.php` で設定確認 → 実際にトリガー発火で動作確認

### 電話通知を実装する場合（TODO）

`NotificationService::sendPhoneNotification()` を Twilio REST API で書き換える：

```php
use Twilio\Rest\Client;

private function sendPhoneNotification($recipient, $data) {
    $twilio = new Client(
        env('TWILIO_ACCOUNT_SID'),
        env('TWILIO_AUTH_TOKEN')   // ← .env に追加要
    );
    $twilio->calls->create(
        $recipient->notification_data,    // 宛先電話
        env('TWILIO_PHONE_NUMBER'),       // 発信元
        ['twiml' => '<Response><Say language="ja-JP">...</Say></Response>']
    );
}
```

### メール通知を実装する場合（TODO）
