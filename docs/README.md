## システム一言まとめ

株式会社アキオカの工場受付に設置したタブレット端末から、**来訪者・納品業者・集荷業者・面接者の受付処理**を行い、担当者へ**Teams・電話（Twilio）・メール**で自動通知するWebアプリケーション。加えて**事前アポイント登録**・**会議室予約（Outlook連携）**・**通知設定**・各種マスタ管理の**管理画面**を備える。

- バックエンド: Laravel 9.52 (PHP 8.0.2+)
- フロントエンド: Vue 3.2 + Inertia.js + Vite 4 + Tailwind 3
- DB: MariaDB（`akioka_db` を生産管理系システムと共有）
- 本番: `133.117.75.136` / `akioka-reception.cloud`（Apache + mod_php）

---

## 主要機能カテゴリ（8分類）

1. **来訪者受付** — 一般来訪者（QR / 手動入力）
2. **アポイント受付** — QR / 4桁受付番号でチェックイン
3. **アポなし来訪者受付** — 部署選択 → 通知
4. **面接受付** — 担当者呼出（Teams通知 / Twilio電話）
5. **納品受付** — 納品書・受領書の撮影／電子印／発注紐づけ
6. **集荷受付** — 集荷伝票の撮影／電子印
7. **施設予約** — 会議室の予約・タイムライン表示（Outlook双方向同期）
8. **管理画面** — アポ・スタッフ・お知らせ・通知設定・施設・プロジェクトグループの管理

## 主要な実装上の注意点（詳細は各ドキュメント参照）

- **担当者の二重表現**: `staff_member_id` は文脈により `users` 参照（visitors / appointments）と `staff_members` 参照（deliveries / pickups）に分かれる。→ [02](./02_データベース設計.md)
- **管理画面の認証は平文パスワード比較・admin判定なし**。デバッグ用エンドポイントも残存。→ [05](./05_認証とセキュリティ.md)
- **通知の実装はTeams（Adaptive Card）のみ完成**。email/phoneはログ出力スタブ。→ [03](./03_機能仕様書.md)
- **DBスキーマとSQLダンプに差異**: スケジュール／プロジェクト系6テーブルがダンプ未反映。→ [02](./02_データベース設計.md)
- **`akioka_db` は外部システムと共有**: `users` / `groups` / `initial_orders` / `stocks` 等は本アプリのマイグレーション外。
