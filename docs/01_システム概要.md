# 01. システム概要

## 1.1 システムの目的

株式会社アキオカの工場受付に設置されたタブレット端末から、来客・物流・面接の受付処理を一元化し、担当者への通知を自動化する。受付業務の無人化・記録のデジタル化を目的とする。

## 1.4 全体アーキテクチャ

```
┌─────────────────────────────────────────────────────────┐
│  受付タブレット（タッチ操作 / 認証なし）                       │
│  Home → 来訪者 / アポイント / アポなし / 面接 / 納品 / 集荷     │
└──────────────────────────┬──────────────────────────────┘
                           │ Inertia (Vue3)
                           ▼
┌─────────────────────────────────────────────────────────┐
│  Laravel 9 （Controllers → Models → akioka_db）            │
│                                                           │
│   ┌── NotificationService / TeamsNotificationService ──┐  │
│   │   trigger_event で通知設定を検索し配信               │  │
│   └─────┬──────────┬──────────┬─────────────────────┘  │
│         ▼          ▼          ▼                          │
│      Teams       Email      Twilio（電話/SMS）            │
│   (実装済)     (スタブ)    (別系統)                       │
│                                                           │
│   ┌── OutlookCalendarService ──┐  ┌── PrintServerService ┐│
│   │   会議室予約の双方向同期      │  │  QR印刷(localhost:8080)││
│   └────────────────────────────┘  └────────────────────┘│
└──────────────────────────┬──────────────────────────────┘
                           │ localstorage.auth ミドルウェア
                           ▼
┌─────────────────────────────────────────────────────────┐
│  管理画面 /admin/*（AdminLayout / localStorage認証）        │
│  ダッシュボード / アポ / スタッフ / お知らせ / 通知設定 /      │
│  施設 / 施設予約 / プロジェクトグループ / 納品 / 集荷         │
└─────────────────────────────────────────────────────────┘
```

### レイヤ構成

- **画面層（Vue / Inertia）**: `resources/js/Pages/` 配下。レイアウトは用途で4種に分岐（下表）。
- **制御層（Controllers）**: `app/Http/Controllers/`。公開受付系・`Admin/`・`Api/`・`Auth/` に区分。
- **サービス層**: `TeamsNotificationService` / `NotificationService` / `OutlookCalendarService` / `PrintServerService` / `QrCodeService`。
- **モデル層（Eloquent）**: `app/Models/`。全モデルがデフォルト接続（`akioka_db`）を使用。

### Layout（画面レイアウト）の使い分け

| Layout | 用途 | 認証 |
|--------|------|------|
| `ReceptionLayout` | 来訪者向けタッチ受付端末画面 | なし |
| `AdminLayout` | `/admin/*` 管理画面 | localStorage認証 |
| `GuestLayout` | ログイン・登録・パスワードリセット | guest |
| `AuthenticatedLayout` | Breeze標準プロフィール画面 | auth（セッション） |
| （なし） | Welcome / FacilityTimeline / Visitor/Complete | なし |

## 1.5 外部サービス連携サマリ

| サービス | 連携箇所 | 実装状況 |
|---------|---------|---------|
| Microsoft Teams | 来訪・納品・集荷・面接の各通知 | **実装済**（Incoming Webhook + Adaptive Card、`<at>email</at>` メンション） |
| Twilio Voice | 面接者の担当者呼出（ブラウザ通話） | 実装済（`TwilioVoiceController`、JWT AccessToken） |
| Twilio SMS/Call | テスト送信 | 実装済（`TwilioTestController`、※SSL検証無効・開発用） |
| Outlook | 会議室予約の重複チェック・作成・更新・削除 | 実装済（`OutlookCalendarService`、失敗時はログのみ） |
| Email | アポイント確認メール / 各種通知 | アポ確認メール（`AppointmentConfirmation`）は実装済。通知のemailチャネルは**ログ出力スタブ** |
| Socket.IO | 外部スキャンツールとのリアルタイム連携 | フロント（`socket.io-client`）のみ。コントローラ層に直接呼び出しなし |
| 印刷サーバー | QRコードの印刷（`http://localhost:8080`） | 実装済（`PrintServerService`） |
