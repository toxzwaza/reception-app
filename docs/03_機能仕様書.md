> 各機能について「画面 → コントローラ → 読み書きするテーブル・カラム → 外部連携」を記載。
> 凡例: **W** = 書き込み（create/update）、**R** = 読み取り。

## 1. 来訪者受付（一般）

**画面**: `Home/Index`（受付トップ）→ `Visitor/ScanQr` または `Visitor/Create` → `Visitor/Complete`
**コントローラ**: `VisitorController`
**連携**: Teams（`TeamsNotificationService::sendCheckinNotification`）

| アクション | ルート | 処理 | テーブル・カラム |
|-----------|--------|------|----------------|
| `scanQr` | GET `/visitor/scan-qr` | QRスキャン画面表示 | - |
| `checkIn` | POST `/visitor/check-in` | QRデータから来訪者作成＋Teams通知 | **W** `visitors`: company_name, visitor_name, staff_member_id, check_in_time(=now) |
| `create` | GET `/visitor/create` | 手動入力フォーム | **R** スタッフ一覧 |
| `store` | POST `/visitor/store` | 手動入力保存＋名刺画像保存＋Teams通知 | **W** `visitors`: company_name, visitor_name, phone, business_card_image, staff_member_id, check_in_time(=now) |
| `complete` | GET `/visitor/complete` | 完了画面 | - |

**バリデーション**（store）: company_name/visitor_name `required|max:255`、phone `nullable|max:20`、business_card_image `nullable|image|max:5120`、staff_member_id `required|exists`。
**名刺画像**: `store('business_cards','public')` で保存。
**通知payload**: reception_number, company_name, visitor_name, staff_member_name, staff_member_email（メンション対象）。

## 2. アポイント受付（公開）

**画面**: `Appointment/Index` → `Visitor/Complete`
**コントローラ**: `AppointmentController`
**連携**: `NotificationService::sendAppointmentNotification`

| アクション | ルート | 処理 | テーブル・カラム |
|-----------|--------|------|----------------|
| `index` | GET `/appointment` | 受付画面 | - |
| `checkInQr` | POST `/appointment/check-in-qr` | QRデータ(JSON)→`appointments`照合→`visitors`作成→通知 | **R** `appointments`、**W** `visitors`: company_name, visitor_name, staff_member_id, visitor_type(='appointment'), check_in_time |
| `checkInNumber` | POST `/appointment/check-in-number` | 4桁受付番号→未チェックインの`appointments`照合→`visitors`作成→通知 | **R/W** `appointments`: is_checked_in(=true), checked_in_at(=now)、**W** `visitors`: 上記＋reception_number |

**バリデーション**: `checkInQr` は qr_data `required|string`、`checkInNumber` は reception_number `required|size:4|regex:/^[0-9]{4}$/`。
**通知**: `NotificationService::sendAppointmentNotification('visitor_checkin', $data, $mentionData)`。`$data` = type, reception_number, company_name, visitor_name, staff_member_name, check_in_time(`Y年m月d日 H:i`), appointment_info。`$mentionData` = mention_id(スタッフemail), email, phone_number(空)。

## 4. 面接受付

**画面**: `Interview/Index`（担当者連絡先表示・呼出）/ `Twilio/VoiceCall`（ブラウザ通話）
**コントローラ**: `InterviewController` / `TwilioVoiceController` / `TwilioTestController`
**連携**: Teams（`notifyInterviewArrival`）、Twilio Voice（JWT）

### InterviewController

| アクション | ルート | 処理 | テーブル・カラム |
|-----------|--------|------|----------------|
| `index` | GET `/interview` | `notification_settings`(trigger_event=`visitor_checkin`, is_active=true) → `activeRecipients`(notification_type=`phone`) から電話リスト構築。**GET時は通知を発火しない**（暴発防止） | **R** `notification_settings`, `notification_recipients`（notification_data=電話番号, user.name=担当者名） |
| `notifyStaff` | POST `/interview/notify-staff` | `notifyInterviewArrival()` でTeams通知 → JSON | （通知のみ） |

> 暴発防止: 過去に画面表示（GET）で通知が発火する不具合があり、POST時のみ通知する設計に修正済み（コミット `e5cf078` 参照）。

### TwilioVoiceController（ブラウザ音声）

| アクション | ルート | 処理 |
|-----------|--------|------|
| `index` | GET `/twilio-voice` | 通話画面 |
| `generateToken` | POST `/twilio-voice/token` | AccessToken(TTL3600)+VoiceGrant発行 → JSON |
| `handleOutgoingCall` | POST `/twilio-voice/outgoing` | 発信TwiML（dial→number） |
| `handleIncomingCall` | POST `/twilio-voice/incoming` | 着信TwiML（dial→client） |
| `callStatusCallback` | POST `/twilio-voice/status` | CallSid/Status/Durationログ |
| `testDevice` | GET `/twilio-voice/test` | env設定有無チェック |

env: `TWILIO_ACCOUNT_SID` / `TWILIO_API_KEY` / `TWILIO_API_SECRET` / `TWILIO_TWIML_APP_SID` / `TWILIO_PHONE_NUMBER`。DBアクセスなし。

### TwilioTestController（発信・SMSテスト）

`/twilio-test`・`/twilio-test/make-call`・`/twilio-test/check-status`・`/twilio-test/send-sms`。env: `TWILIO_ACCOUNT_SID` / `AUTH_TOKEN` / `PHONE_NUMBER`。**SSL検証無効（開発用）**。DBアクセスなし。

## 5. 納品受付（納品書・受領書）

## 通知システムの仕組み（横断）

```
受付イベント発生（来訪/納品/集荷/面接）
   │
   ▼
NotificationService::sendNotification(trigger_event, data, mentionData)
   │  trigger_event で notification_settings を検索（is_active=true）
   ▼
notification_recipients を notification_type 別に取得
   ├── teams  → TeamsNotificationService::notify()  【実装済】
   │            Incoming Webhook + Adaptive Card v1.0
   │            <at>{email}</at> でメンション
   ├── email  → ログ出力のみ（スタブ・未実装）
   └── phone  → ログ出力のみ（スタブ・未実装。発信はTwilio別系統）
```

- **Teams Webhook URL**: `config('services.teams.webhook_url') ?: env('TEAMS_WEBHOOK_URL')`。`Http::timeout(10)->withOptions(['verify'=>false])` でPOST。
- **メンション**: `notification_data` に格納された email を `<at>email</at>` 形式で `msteams.entities` に設定。
- 詳細は `../通知システム仕様書.md` も参照。

> **改修時の注意点まとめ**
> - `delivery_type` のバリデーション値が capture/store で不一致
> - `sealed_at` が Delivery/Pickup の `$fillable` 未登録（update時に注意）
> - `unlinkOrder` は `receive_flg`/`receipt_flg` を元に戻さない
> - email/phone 通知は未実装スタブ
> - Twilio・TwilioTest は SSL検証無効（開発用設定が残存）
