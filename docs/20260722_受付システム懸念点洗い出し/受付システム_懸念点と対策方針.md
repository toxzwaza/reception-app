# 受付システム 社内展開に伴う懸念点と対策方針

作成日：2026-07-20　／　担当：システム開発部門
対象システム：受付システム（reception-app）
根拠資料：`カウンター業務のヒアリング.xlsx`（記入者：野島・田村・加藤／記入日 2026-07-06〜07-13）

---

## 1. 背景と目的

新社屋移転に伴い、**現在の有人カウンター業務が無人受付（受付タブレット reception-app）に置き換わる**。
これにより「これまで人が その場で判断・案内・受け渡しをしていた業務」がどうなるか、現場から **22件の懸念** が挙がった。

本資料は、22件を **①受付システムで対応（機能実装）／②運用ルールで対応／③物理設備で対応** に切り分け、
**どの対策機能を実装するかを意思決定する**ためのたたき台である。最終判断は村上が行う。

---

## 2. 全体傾向

- 懸念22件のうち、**大半は「イレギュラーな来訪・配達・伝票のやり取り」**で、その場に人がいれば口頭で片付いていたもの。
- 現場が記入した対応案を見ると、**多くが「旧⇔新事務所の内線」「玄関・裏口の内線で総務／該当部署を呼ぶ」＝人による対応** に帰結している。
- つまり無人受付化の肝は、**「非定型の用件を、正しい相手（総務・技術・該当部署）へ確実につなぐ導線」** をどう用意するか。
- 純粋にシステム機能で解決・軽減できるものは限定的だが、**「呼び出し」「場所案内」「受領記録」** の3点は効果が大きい。

## 3. 受付システム（reception-app）で対応する機能案

懸念を機能単位に束ねた。**優先度：高＝無人化の成立に必須／中＝実務効率・記録／低＝稀で運用可**。

### 【F1】部署別・総務 呼び出し導線の拡充　★優先度：高
- **狙い**：無人受付でも「非定型の用件」を正しい相手へつなぐ受け皿。多くの「内線で対応」懸念を包括。
- **内容**：受付画面に「ご用件が上記にない方／業者・配達の方」等の導線を追加し、**総務・技術など該当部署へTeams通知＋内線案内**。担当不在時は**複数名／グループ通知にフォールバック**。
- **対応する懸念**：No.7・8・11・12・13・18・21（＋来客系 6・10）
- **既存資産**：部署電話番号・通知設定・Teams通知・電話発信(Twilio) が流用可能。

### 【F2】納品・集荷の「場所案内／地図表示」　★優先度：高
- **狙い**：これまで人が口頭でしていた「どこに置くか／製品がどこか」を画面で案内。
- **内容**：納品・集荷の受付画面で、**注文先・品目に応じた置き場所や構内地図を表示**。注文書FAX時に地図を同送（運用）と併用。
- **対応する懸念**：No.3（納品場所の説明）・No.13（ドライバーが製品場所不明）

### 【F3】受領・伝票の電子化／記録　優先度：中
- **狙い**：記録が必要な受け渡しをデジタル化（QR・電子印は既存機能）。
- **内容**：**引取・受領時に品名/数量を記録**、電子受領。ただし「朝早い／相手が押して帰る」ものは物理運用の継続が現実的。
- **対応する懸念**：No.17（光塗装の仮伝票）・No.19（中子の受領）※No.4・20・21・22は物理運用が妥当

### 【F4】配達品の受付・保管案内　優先度：中
- **狙い**：配達物の「置き場所」「受領」の案内を画面表示。
- **内容**：配達受付で**置き場所（要冷蔵・専用容器など）を案内**、受領印運用と連動。
- **対応する懸念**：No.1（インク）・No.2（要冷蔵）・No.12（着払い）・No.16（郵便）※冷蔵庫等は物理

### 【F5】来訪類型の追加（アポなし来客・業者）　優先度：低
- **狙い**：挨拶来客・点検業者などの受付導線。ただし**稀なため F1 の総務呼び出しで代替可**。
- **対応する懸念**：No.6・10（来客）・No.14・15（業者）

## 4. 運用ルール・物理設備で対応する項目（システム対象外）

システムでは解決せず、**運用ルールや設備**で対応すべきもの。※現場の対応案が既に妥当なものが多い。

| 区分 | 懸念No. | 対応（現場案ベース） |
|---|---|---|
| 物理設備 | 1・2 | 使用済みインク用の専用容器、配達品置き場に小型冷蔵庫を設置 |
| 運用ルール | 4・9 | 産廃マニフェスト・劇物譲受書はカウンター上でドライバー記入／押印返送 |
| 運用ルール | 5・16 | クロネコ発送・郵便は集荷場所へ／配達員が受領印、または内線で総務 |
| 運用ルール | 14・15 | 業者の鍵受け渡しは総務対応（来社日にカウンターへ鍵）、記帳は要否を総務確認 |
| 運用ルール | 20・22 | 朝早い集荷（福熱・ティエラ）の伝票は現状どおり旧事務所カウンターに設置 |
| 設備（内線） | 7・8・11・12・13・18・21 | **旧⇔新事務所・玄関・裏口の内線整備が前提**（F1 呼び出しの土台） |

> ⚠ **前提条件**：F1（呼び出し）を活かすには、**内線設備（旧⇔新事務所／玄関／裏口）の整備が必須**。これはシステム外の設備投資判断。

## 6. 意思決定のポイント（村上判断用）

1. **無人受付の基本コンセプトを確認する**
   - 「定型（アポ来訪・納品・集荷）はシステムで完結、非定型（イレギュラー・来客・業者）は**呼び出し→総務／該当部署**」で合意できるか。

2. **実装する機能を選ぶ**（優先度順）
   - ✅ **F1 呼び出し導線**（高）… 22件中の多くを受ける“受け皿”。**最優先で実装推奨**。
   - ✅ **F2 場所案内／地図**（高）… 納品・集荷の実務に直結。
   - ⬜ **F3 受領・伝票の電子化**（中）… 記録が必要な No.17・19 に限定して実装。
   - ⬜ **F4 配達品の受付・保管案内**（中）… 効果は限定的。運用併用。
   - ⬜ **F5 来客・業者の受付類型**（低）… 稀。F1で代替可、当面は運用でも可。

3. **前提となる設備を確認する**（システム外だが必須）
   - 旧⇔新事務所・玄関・裏口の**内線整備**、配達品置き場・小型冷蔵庫・専用容器。

4. **運用で確定する項目を切り分ける**
   - マニフェスト・劇物押印・鍵受け渡し・朝早い集荷伝票などは**運用ルール文書化**で対応（システム開発不要）。
